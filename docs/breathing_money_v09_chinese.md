# [预发布][研究] 呼吸货币 — PoW自适应货币基础层

**没有募资，没有预挖，没有上线时间表，没有代码。**
**这是一份请求对抗性审查的研究草案，在任何实现之前发布。**

我试图证伪三个假说:

1. 链上结算需求可以在不依赖预言机的情况下作为供给信号。
2. 一个奖励控制器可以在不冻结货币政策的前提下保障矿工永久报酬。
3. 闭环系统（供给注入 ↔ 挖掘提取 ↔ 算力 ↔ 出块时间）在现实条件下可以稳定运行。

如果你能打破其中任何一个，那对我非常有价值。

---

## 本帖讨论范围

本帖请求PoW社区攻击一件事: **一个PoW L1货币基础层的自适应发行与挖掘机制。**

讨论范围内:
- 供给信号（什么驱动发行）
- 挖掘控制器（什么决定区块奖励）
- 下限保障（矿工报酬，池子枯竭）
- 区块空间策略（作为货币安全机制）
- 难度调整假设

讨论范围外:
- 支付体验，商户接受度，"人们会不会把它当现金用"
- L2架构，隐私，可替代性
- 通胀货币在哲学上对不对
- 比特币的货币政策需不需要修改

这些都是真问题，但不是本帖的问题。如果货币基础层机制通不过对抗性审查，建在上面的任何东西都没有意义。

## 我们不知道什么

设计之前，先说清楚我们不确定的地方。

**我们不知道链上交易量是不是一个好的供给信号。** 它是我们找到的唯一纯链上、不需要预言机、不需要人为输入、且不投入真实价值就无法产生的信号。它不完美——钱包行为、交易所批处理、L2结算模式、托管方操作都会污染它。**如果你知道一个更好的信号，满足以上所有约束条件，请提出来。** 我们考察过的每一个替代方案（交易笔数、手续费加权交易量、UTXO集合大小、手续费燃烧）要么攻击面更大，要么违反设计原则。

**我们不知道4%锚点是不是对的数字。** 它参考了长期美国经济数据（实际GDP年化约3.2-3.5%，跨越100-200年；实际股票价格增值约4%）。我们选了区间上限，把系统偏向花钱而非囤币。**如果3%或5%更适合这个设计目标，我们想听你的理由。**

**我们不知道所有攻击向量。** 下面的分析描述了我们认为能使持续操纵无利可图的激励结构。**如果你能构造一个在这些参数下持续盈利的操纵场景，请分享。** 这是真正有价值的贡献。

**我们不知道200个月的MA窗口是不是正确的权衡。** 越长 = 越难操纵，越慢适应。越短 = 越敏感，越容易被博弈。**如果你能设计一个更短且操纵阻力相当的窗口，请展示给我们。**

**我们不知道商家是否会接受一种供给持续增长的货币。** 基础层激励花钱，但无法激励收钱。商家选择接受、持有还是立即兑换，是涌现层的结果，我们不定义也不预测。

## 什么会推翻这个设计

如果以下任何一条被证明成立，这个设计应当被放弃或彻底重写:

1. **存在持续盈利的攻击。** 有人构造出一个现实的自我刷量或卡特尔场景，在给定的区块空间和手续费参数下，12个月以上持续正收益。

2. **下限不兼容。** 在满足所述约束条件的情况下，下限机制的提取仍然超过注入——特别是在月度边界、奖励取整或离散出块时序上。

3. **不存在可接受的DAA。** 在共享哈希算法（SHA-256）条件下配合月度奖励更新，没有难度调整算法能在现实算力迁移事件中维持稳定出块时间。

4. **复杂度不值得。** 简单的固定尾部发行（Monero式）在更少的攻击面和更低的机制复杂度下提供了等效或更好的特性。如果自适应组件增加了风险却没有带来相应收益，更简单的设计胜出。

5. **信号不可修复地嘈杂。** 即使有200个月的平滑，链上交易量仍被非经济活动主导，不携带关于真实结算需求的可用信息。

## 背景: 为什么做这个

中本聪的论文标题是"比特币: 一种点对点电子现金系统"。它的货币政策——固定上限、减半——把它变成了数字黄金。这不是采用率的失败，而是通缩货币的理性结果。

本项目尝试设计一个货币基础层，让花钱成为理性行为而非非理性行为。它不自称是电子现金本身——它为现金在L2上的涌现提供条件。现金是否真的涌现，不是任何基础层能保证的。

这条链不和比特币竞争。比特币是储值资产，这是交换媒介基础层。如果人们花这个币、存比特币，两个系统都在按预期运作。

**理论基础:** SAE（以自身为目的）框架的三篇论文为本设计提供了方法论:

- 凿构循环——如何设计承认自身局限性的系统（[方法论总论, DOI: 10.5281/zenodo.18842450](https://doi.org/10.5281/zenodo.18842450)）
- 任何形式化都产生不可消除的余项——为什么本设计必须包含常数且无法达到完美（[ZFCρ第一篇, DOI: 10.5281/zenodo.18914682](https://doi.org/10.5281/zenodo.18914682)）
- 基础层/涌现层区分——为什么货币基础层提供条件而不预设结果（[SAE第一篇, DOI: 10.5281/zenodo.18528813](https://doi.org/10.5281/zenodo.18528813)）

评估这个设计不需要阅读以上任何论文。它们在这里是给想理解背后思路的人准备的。完整存档: [self-as-an-end.net](https://self-as-an-end.net)

**关于作者:** 计算机科学博士，2009年读过比特币白皮书，目前从事区块链行业（合规代币化）。没有预挖，没有创始人分配，没有早期优势。创世封存（见下文）在结构上保证我和其他所有人从同一规则起步挖矿。

## 当前工作假设

以下所有内容都是工作假说，不是最终决定。全部可以根据本次讨论和后续测试网结果进行修改。

### 创世

- 初始总量: **2100万** （和比特币相同，但作为起点而非上限）
- **创世封存:** 110万币发送到不可花费地址。协议层强制执行，无私钥。致敬中本聪约110万从未动过的BTC——但这里是代码保障，不是意志力。永久封死，链上可见。
- **模拟历史:** 创世区块包含一条按4%年化增长模拟的交易量历史曲线，作为移动平均的基线。
- **第一阶段（第0-199月）:** 供给增长率锁定为4%。真实数据开始积累，自适应公式尚未激活。
- **第二阶段（第200月起）:** 公式激活，MA使用200个月的真实数据。模拟数据逐月替换，至第400月完全使用真实数据。

我们知道200个月很长。这是一个取舍: 稳健优先于响应速度。

### 第一凿: 供给函数

决定每月有多少新供给进入待挖池。

第一阶段: `new_supply = total_supply × 0.04 / 12`

第二阶段:
```
annual_growth_rate = max(tail_floor, (MA_current / MA_200months_ago)^(12/200) - 1)
new_supply = total_supply × annual_growth_rate / 12
```

- MA = 非coinbase交易输出值的200个月移动平均
- tail_floor = 最低年增长率（候选值: 0.5%/年），确保永续发行

**信号:** 追踪的是L1结算需求，不是直接的"人类经济活动"。不完美的代理变量，之所以用它是因为在免信任约束下它是唯一合法的选择。

**排除coinbase**（防止自反性）。**包含找零输出**（放大了绝对交易量但在增长率的比值中大致抵消——当钱包/批处理行为改变时这个假设会弱化）。

**没有上限。** 200个月MA本身就是巨大的阻尼器。不设人为天花板。

**承认顺周期性。** 币值下跌时，同一笔真实交易需要更多本币，名义交易量上升。200个月MA将此效应压缩到极低的频率，但不消除。这是一个已知局限。

### 第二凿: 挖掘函数

决定每个区块矿工能提取多少币。

**难度（出块时间）:** 目标10分钟。当前占位符: 比特币式2016区块重新调整。**我们承认这对共享ASIC算法的少数链很可能是致命的**——矿池算力跳跃可以崩溃出块时间，算力撤离后链被冻结。几乎可以确定需要逐块或连续的DAA（如ASERT、LWMA、DigiShield）。也可能需要抗ASIC算法（如RandomX）以实现公平的早期分发。**这不是神圣不可变的选择——欢迎提出替代的DAA和算法方案。**

### 签名方案: 从创世开始后量子

本链从创世区块起使用ML-DSA（CRYSTALS-Dilithium, FIPS 204）。**没有椭圆曲线密码学，没有secp256k1。**

2026年3月30日，Google Quantum AI发布了破解secp256k1上256位椭圆曲线离散对数问题的资源估算（Babbush等，"Securing Elliptic Curve Cryptocurrencies against Quantum Vulnerabilities"）。结论: Shor算法可以在约1,200个逻辑量子比特配合约9,000万Toffoli门，或约1,450个逻辑量子比特配合约7,000万Toffoli门的条件下执行。在当前错误率的超导架构上，这意味着不到50万个物理量子比特，比此前估计降低了约20倍。论文引入了"on-spend"攻击的概念: 量子攻击者在交易确认之前就从公钥推导出私钥。

比特币已经在行动。BIP-360（Pay-to-Merkle-Root）已合并进比特币官方BIP仓库，移除了Taproot中量子脆弱的keypath spend。BTQ Technologies已在测试网上部署了包含Dilithium后量子签名操作码的可用实现。但BIP-360只解决long-range攻击（暴露的静态公钥），不解决on-spend攻击。而且BIP-360联合作者Ethan Heilman估计，即使乐观情况下，比特币完成完整迁移也需要大约七年。

**这条链还不存在，这是一个优势。** 我们不需要在活跃网络上改造量子抗性，从第0个区块就建入。

设计选择:

1. **ML-DSA-65（Dilithium3）作为创世签名方案。** NIST标准化（FIPS 204），保守安全级别，纯格基方案。公钥: 1,952字节，签名: 3,293字节。比ECDSA（33 + 72字节）大得多，但L1是结算层——交易体积可以接受。高频支付属于L2。

2. **算法灵活的地址格式。** 地址携带版本字节，选择签名验证规则。版本0x02 = Dilithium。未来版本可以通过软分叉引入其他PQC方案（FN-DSA/FALCON, SLH-DSA/SPHINCS+）或混合方案，无需改变货币层。

3. **签名方案属于可维护层，不属于货币宪法。** 区块空间策略、供给函数、挖掘函数和下限约束是不可变的。签名算法是工程层面的，它可以随密码学认知的进步而升级。

4. **区块空间影响。** 约3.3 KB的签名意味着1 MB区块约容纳300笔交易，而ECDSA下约3,000笔。这是特性而非缺陷: L1区块空间稀缺性强化了供给信号（更少的交易 = 每笔交易经济意义更大），并提高了交易量操纵的成本。

**求助:** ML-DSA-65、FN-DSA（FALCON-512）和SLH-DSA（SPHINCS+-128s）对一个基于UTXO的PoW链各有什么具体权衡？对于一条尚未上线的链，是否有理由使用混合签名（经典 + PQC），还是直接纯PQC是正确选择？

**区块奖励（挖掘速率）:** 按月调整。

```
block_reward = base_reward × max(spring_floor, spring(actual_pool / planned_pool))
```

- base_reward ≈ 16币/块（校准: 16 × 52,560 ≈ 84万/年 ≈ 2100万的4%，有待模拟验证）
- planned_pool = 从创世开始按4%年化轨迹预测的池子大小
- 候选弹簧函数: `max(spring_floor, x^k)`, k > 0

### 4%锚点

一个指向交换媒介的有意识的宪法偏置，参考了200年美国经济数据（实际GDP 3.2-3.5%，实际股票价格增值约4%）。不是经验定律——是设计选择。弹簧允许系统自由偏离; 4%是弹簧最松弛的位置，不是系统被迫停留的位置。

### 下限兼容性约束

tail_floor和spring_floor必须满足:

```
下限注入 ≥ 下限提取

total_supply × tail_floor / 12 ≥ base_reward × spring_floor × blocks_per_month
```

使用候选值（tail_floor = 0.5%/年，base_reward = 16，约4,380块/月）:

```
8,750 ≥ 70,080 × spring_floor
→ spring_floor ≤ 0.125（最低区块奖励 ≈ 2币/块）
```

硬性数学约束。保证下限机制下的提取永远不超过注入。随着total_supply增长，约束越来越容易满足。

**求助:** 请验证这个计算。找出下限在满足约束条件的情况下仍然失效的场景（月度边界、取整、离散出块时序）。

### 矿工权利

tail_floor + spring_floor = **协议层面永久的矿工报酬保障。** 这不是补贴——是对维持链存在的服务的支付。

比特币将矿工收入转向手续费。如果手续费市场失灵，安全就失灵。本链保证永久的正名义补贴。它**不**保证足够的实际购买力——如果币价崩溃，矿工可能仍然会离开。我们消除的是结构性失败模式（补贴→0），不是市场性失败模式（价格→0）。

### 区块空间作为货币安全

交易量作为供给信号意味着区块空间成本 = 信号安全成本。**区块大小、手续费策略和粉尘规则是货币宪法参数，不是实现细节。**

**向矿工和节点运营者求助:** 什么样的区块重量、最低手续费和粉尘阈值能让操纵成本不可忽略，同时保持L1结算的可负担性?

### 反操纵: 博弈论

**个体:** 接近零边际成本的洗交易。但产生的供给增加按算力比例分配给所有矿工; 操纵者的收益经过200个月稀释后微乎其微。

**集体（囚徒困境）:** 如果所有矿工都刷交易量: 供给增长，所有持仓被稀释，信心下降，价格下跌，难度上升。纳什均衡: 不刷。

**不宣称不可能。** 宣称的是激励结构使持续操纵在有规模的网络中不经济。

**求助:** 构造一个持续盈利的操纵场景。在候选区块空间参数下，将供给信号持续移动1%/5%/10%达12个月需要多少成本?

### 两凿交互

通过待挖池的负反馈。共同决定均衡。现实中的耦合: 奖励 → 利润率 → 算力 → 出块间隔（直到难度重新调整）→ 提取 → 池子 → 下一期奖励。

**这个闭环在主网前需要稳定性仿真。** 没有稳定性分析我们不会上线。

### 冷启动奖励

前12个月: `effective_supply = new_supply × (1 + max(0, (12-month)/12))`

第0月: 2倍。第6月: 1.5倍。第12月起: 1倍，永久生效。写入创世代码，不可修改。

### 可扩展性

**L1:** PoW共识、供给函数、挖掘函数、UTXO价值转移。没有智能合约。

**L2:** 其他一切。L1预留最小锚定接口（测试网之后定义规格）。

### 协议维护

**不可变层（货币宪法）:** 供给函数+下限，挖掘函数+下限+兼容性约束，区块空间策略，创世封存，冷启动规则，矿工权利，分叉权。

**可维护层（工程实现）:** P2P网络、存储、客户端、非经济性安全补丁。

**没有基金会。没有特权维护者。**

### 分叉权

宪法不可修改但不神圣。任何人都可以分叉。我不反对分叉。我没有因分叉而受损的经济利益。

## 这不是什么

不是算法稳定币（没有锚定，没有重新基准，没有预言机）。不会遭受补贴死亡螺旋（结构性补贴归零风险已消除; 市场风险仍在）。不和比特币竞争。从创世开始后量子（ML-DSA/Dilithium，没有secp256k1——本设计不携带椭圆曲线风险）。最近的先例: Monero的尾部发行——本链进一步引入了自适应发行 + 自适应挖掘 + 数学化的下限兼容约束 + 后量子签名。

## 我需要帮助的地方

1. **更好的供给信号** — 纯链上，无预言机，无人为输入，操纵成本不低于交易量。一个值得探索的方向: **手续费燃烧率作为信号**（EIP-1559式）。燃烧的手续费是不可回收的成本，能解决零成本操纵问题。待解决的问题是如何在无人为输入的情况下确定燃烧率，同时确保系统保持净通胀（注入 > 燃烧）。如果你有这方面的设计，我们想看到。
2. **弹簧函数形式和k值** — 常数还是自适应?
3. **下限值** — 0.5%/年的tail，≤0.125的spring floor，对吗?
4. **区块空间参数** — 重量、手续费、粉尘阈值。信号安全 vs 可用性。
5. **DAA和哈希算法** — 2016区块重新调整对少数链很可能是致命的。几乎可以确定需要逐块DAA（ASERT/LWMA/DigiShield）。抗ASIC算法（RandomX）用于公平早期分发? 什么组合最适合这个经济模型?
6. **闭环稳定性建模** — 奖励 ↔ 算力 ↔ 出块时间 ↔ 挖掘。如果你有控制系统经验，我想和你合作。
7. **200个月窗口** — 太长? 给我看一个更短且操纵阻力相当的窗口。
8. **攻击场景** — 量化的、持续的、带成本估算。
9. **自适应组件是否值得它的复杂度（对比简单的固定尾部发行）?**
10. **冷启动形状** — 当前设计从2倍开始衰减。是否应该从低到高爬坡到峰值，以防止早期大算力抢挖? 什么爬坡形状?
11. **PQC签名方案选择** — ML-DSA-65 vs FN-DSA vs SLH-DSA对UTXO PoW链的权衡？约3.3 KB签名的区块重量影响？混合签名还是纯PQC？
